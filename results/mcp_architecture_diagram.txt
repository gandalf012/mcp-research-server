==================================================================================================
                    LIFECYCLE SEQUENCE — HOW A SESSION WORKS
==================================================================================================

  MCP CLIENT                                         MCP SERVER
  ----------                                         ----------
       |                                                  |
       |  1. INITIALIZE (capability negotiation)          |
       | ------------------------------------------------> |
       | <------------------------------------------------ |
       |                                                  |
       |  2. DISCOVER primitives                          |
       |     tools/list  --------------------------------> |
       |     resources/list  ----------------------------> |
       |     prompts/list  ------------------------------> |
       | <------------ (lists returned) ----------------- |
       |                                                  |
       |  3. OPERATE                                      |
       |     tools/call  --------------------------------> |
       |     resources/read  ----------------------------> |
       | <------------ (results returned) --------------- |
       |                                                  |
       |  4. NOTIFICATIONS (real-time, no response)       |
       | <-------------- (server pushes updates) -------- |
       |                                                  |
       |  5. TERMINATE                                    |
       | ------------------------------------------------> |
       |                                                  |

==================================================================================================
                    PRIMITIVES — QUICK REFERENCE
==================================================================================================

  SERVER PRIMITIVES                    CLIENT PRIMITIVES
  +-----------+-----------------------+  +-----------+-----------------------+
  | Primitive | Purpose               |  | Primitive | Purpose               |
  +-----------+-----------------------+  +-----------+-----------------------+
  | Tools     | Execute actions       |  | Sampling  | Request LLM output    |
  | Resources | Provide context data  |  | Elicit.   | Request user input    |
  | Prompts   | Interaction templates |  | Logging   | Send debug messages   |
  +-----------+-----------------------+  +-----------+-----------------------+

  TRANSPORT COMPARISON
  +---------------------+-------------------------------+---------------------------+
  | Transport           | Use Case                      | Auth                      |
  +---------------------+-------------------------------+---------------------------+
  | Stdio               | Local / same-machine process  | N/A (process-level)       |
  | Streamable HTTP     | Remote / network server       | Bearer, API Key, OAuth    |
  +---------------------+-------------------------------+---------------------------+
